## 三、ProactiveScheduler

### 3.1 职责与位置

| 项目 | 说明 |
|------|------|
| **新模块** | `src/daemon/proactive_scheduler.rs` |
| **职责** | 协调 CEO 周期、实体 idle 巡逻、Cron、Heartbeat；在 daemon 内替代「仅 gateway + channels + heartbeat + cron」的并行 spawn，改为由 Scheduler 统一调度「下一该谁跑」 |
| **输入** | Config（proactive.*）、EntityPool、MessageBus、ConversationBus、Cron 下次触发时间、Heartbeat 下次触发 |

### 3.2 CEO 主动循环

| 项目 | 说明 |
|------|------|
| **周期** | `proactive.ceo_interval_secs`（默认 300）；或由「实体汇报」事件触发一次 |
| **逻辑** | 调用 agent run，target_entity_id=ceo；system prompt 或工具中注入：收件箱中的汇报、待审批项、instance_status；CEO 产出决策（assign_task、approve 等） |
| **与 Cron** | 可保留 `JobType::Agent target=ceo` 的 cron 条目，与 ProactiveScheduler 的 CEO 周期二选一或并存（建议先并存，cron 作为补充） |

**单元测试**：

- `proactive_scheduler::tests::ceo_tick_invoked_at_interval`
- `proactive_scheduler::tests::ceo_tick_triggered_by_report_event`

### 3.3 实体主动循环

| 项目 | 说明 |
|------|------|
| **任务驱动** | 实体被调度时：先 `inbox_recv(entity_id)`，若有任务则执行 agent run（target=该 entity），执行完可标记完成或汇报 |
| **巡逻** | `proactive.entity_idle_interval_secs`（默认 60）：无任务时执行「巡逻」prompt（自检、可发起 conversation 等） |
| **并发** | `proactive.max_concurrent_turns`（默认 3）：同时最多 N 个实体在跑 turn |

**单元测试**：

- `proactive_scheduler::tests::entity_poll_inbox_and_runs_agent_when_task_exists`
- `proactive_scheduler::tests::entity_idle_runs_patrol_prompt`
- `proactive_scheduler::tests::respects_max_concurrent_turns`

### 3.4 配置

| 配置项 | 默认 | 位置 |
|--------|------|------|
| proactive.ceo_interval_secs | 300 | schema.rs |
| proactive.entity_idle_interval_secs | 60 | schema.rs |
| proactive.ceo_enabled | true | schema.rs |
| proactive.entity_autonomous | true | schema.rs |
| proactive.max_concurrent_turns | 3 | schema.rs |

**单元测试**：`config::schema::tests::proactive_config_defaults`

### 3.5 Daemon 集成

| 位置 | 修改 |
|------|------|
| `src/daemon/mod.rs` | 若 `config.instance.is_some()` 且 proactive 启用，则 spawn `ProactiveScheduler::run(config, entity_pool, message_bus, conversation_bus, ...)`，与 gateway/channels/heartbeat/cron 并行；否则保持现有 spawn 逻辑 |

**单元测试**：

- `daemon::tests::when_instance_and_proactive_spawns_scheduler`
