# 改造实施计划 — 阶段 2.5：技能与 ClawHub

> 依赖：阶段 2（Entity、技能白名单已存在）。实现技能加载作用域、ClawHub 工具、平台扫描、管理员审批全局技能。每模块需单元测试。

## 一、技能加载路径与优先级

### 1.1 当前行为（对照）

| 位置 | 当前 | 目标 |
|------|------|------|
| `src/skills/mod.rs` | 从 `workspace_dir/skills/<name>/` 加载；open_skills_dir | 增加「实例 workspace」与「集群 shared」两条路径；优先级：实例 workspace > open_skills > shared |

### 1.2 代码修改

| 项目 | 说明 |
|------|------|
| **配置** | `Config::skills` 或 Config 顶层：可选 `shared_skills_dir: Option<PathBuf>`（集群根 `shared/skills`） |
| **加载逻辑** | `load_skills_with_config(workspace_dir, config)`：按顺序扫描 ① `workspace_dir/skills/*` ② `config.skills.open_skills_dir` ③ `cluster_root/shared/skills/*`；去重时以优先级高为准 |
| **实体白名单** | 已有：EntityRuntime.skills_allowlist；执行时只加载该实体 allowlist 内的技能（过滤已在阶段 2） |

**单元测试**：

- `skills::tests::load_respects_priority_instance_over_open_skills`
- `skills::tests::load_includes_shared_dir_when_configured`
- `skills::tests::entity_allowlist_filters_loaded_skills`

## 二、ClawHub 工具

### 2.1 工具列表

| 工具名 | 行为 | 作用域 |
|--------|------|--------|
| clawhub_search | 搜索技能（关键词/标签） | 只读；可查 instance + global 目录或远程索引 |
| clawhub_browse | 浏览技能列表/详情 | 同上 |
| clawhub_install | 安装技能到实例或申请全局 | instance：直接写入 `workspace/skills/<name>/`；global：写入待审批队列，由管理员审批后写入 `shared/skills/` |

### 2.2 实现位置与接口

| 位置 | 说明 |
|------|------|
| **新文件** | `src/tools/clawhub_search.rs`, `clawhub_browse.rs`, `clawhub_install.rs`（或统一 `src/tools/clawhub.rs` 下三个函数） |
| **参数** | clawhub_install 需 `name`, `source`（url 或 registry id）, `scope: "instance" | "global"`；global 时返回「已提交审批」 |
| **依赖** | 技能扫描器（见 2.3）；审批队列（见 2.4） |

**单元测试**：

- `tools::clawhub_search::tests::returns_results_from_registry_or_fs`
- `tools::clawhub_install::tests::instance_scope_writes_to_workspace`
- `tools::clawhub_install::tests::global_scope_submits_for_approval`

### 2.3 平台强制扫描

| 项目 | 说明 |
|------|------|
| **时机** | clawhub_install 写入前（或从源拉取后）；以及 skill_approve_global 前 |
| **内容** | 路径穿越、危险命令（rm -rf /、eval 等）、非允许域名、脚本注入等；复用或引用 `src/skills/audit.rs` 现有逻辑，扩展规则 |
| **结果** | 不通过则拒绝安装/审批，并返回原因 |

**单元测试**：

- `skills::audit::tests::rejects_path_traversal`
- `skills::audit::tests::rejects_dangerous_shell_commands`
- `tools::clawhub_install::tests::refuses_install_when_scan_fails`

### 2.4 管理员审批全局技能

| 项目 | 说明 |
|------|------|
| **存储** | 待审批列表：`instances/admin/workspace/state/skill_approval_pending.json` 或等价；项含 source、requested_by_instance、name、scan_result |
| **工具** | 仅管理员上下文可用：`skill_approve_global`, `skill_reject_global`（可做 CLI 子命令或 Gateway API） |
| **流程** | approve：再次平台扫描 → 通过则写入 `cluster_root/shared/skills/<name>/`，并从未决列表移除；reject：仅从未决列表移除 |

**单元测试**：

- `tools::skill_approve_global::tests::rejects_when_not_admin`
- `tools::skill_approve_global::tests::writes_to_shared_and_clears_pending`
- `tools::skill_reject_global::tests::removes_from_pending_only`
