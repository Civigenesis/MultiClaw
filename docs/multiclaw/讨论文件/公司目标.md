# 项目上下文信息汇总

**公司品牌**：**Civigenesis**  
**口号**：**Where Intelligence Becomes Society.** —— 让智能体从孤立个体走向有组织、有演化、有文明的集群社会。

## 二、开源项目 MultiClaw 核心特性

**技术栈**：100% Rust，轻量级、快速启动、低内存占用，可在低端硬件（如 $10 开发板，<5MB RAM）上运行。

### 架构设计
- **Trait 驱动架构**：所有子系统（Provider、Channel、Memory、Tool、Runtime 等）均为 trait，通过配置可切换实现。
- **可插拔一切**：模型提供商、通信渠道、工具、内存后端均可替换。
- **五层架构 (v6.0)**：
  - **全局层**：跨实例资源调度与全局策略
  - **编排层**：任务分解、Agent 协调
  - **核心层**：基础能力（记忆、技能、模型路由）
  - **执行层**：具体任务执行与工具调用
  - **可观测层**：全链路监控与日志

### 安全与可靠性
- **安全优先**：配对机制、沙箱隔离、允许列表、速率限制、加密密钥存储。
- **自主恢复能力**：故障检测、检查点管理、自动恢复机制。

### 企业级多实例管理
- 支持多公司/多实例管理，每个实例有独立的董事长 Agent（用户分身）。
- **A2A 跨公司通信**：Agent-to-Agent 协议支持跨实例安全通信。

### 集群状态与看板体系
- **ClusterState**：实时监控所有节点（董事长、公司、团队、Agent）。
- **CompanyManager**：公司/团队管理（创建公司、团队、快速部署）。
- **五层可观测性看板**：
  - **L5 用户看板**：全局摘要、资源概览、建议、告警（面向人类用户）。
  - **L4 董事长看板**：多实例概览、重大事件、成本分析（面向董事长 Agent）。
  - **L3 CEO 看板**：项目列表、待审批、团队排名（面向 CEO Agent）。
  - **L2 团队看板**：任务进度、Worker 状态、知识库（面向团队负责人）。
  - **L1 Agent 看板**：执行记录、健康状态、收件箱（面向 Worker Agent）。

## 三、核心技术模块（待深入设计）

### 1. 智能上下文管理
- **问题**：长时间运行导致模型上下文窗口溢出，关键信息被稀释。
- **方案**：分层记忆架构（工作记忆折叠 + 长期记忆检索）。
  - 工作记忆：滑动窗口 + 关键事件缓存。
  - 记忆折叠：触发条件（窗口满/响应变慢）时，由轻量级折叠器将历史压缩为结构化 JSON 记忆胶囊。
  - 长期存储：向量数据库（如 Chroma）存储胶囊，支持按需检索。
  - 遗忘机制：记忆衰减与重要性评分。

### 2. 技能系统
- **问题**：兼容 OpenClaw 现有生态，同时支持高性能原生技能。
- **方案**：
  - 双引擎兼容：API 兼容层支持 Node.js 技能，Rust 原生 SDK 提供高性能。
  - 沙盒隔离：WASM 或轻量容器隔离技能执行。
  - 技能市场：去中心化技能仓库（版本管理、依赖解析）。
  - 技能监控：内置调用耗时、成功率、资源消耗指标。

### 3. 持续进化机制
- **问题**：智能体集群如何共享经验、整体进化？
- **方案**：
  - 经验提炼：Agent 执行后自动识别成功/失败模式，生成“经验胶囊”（基因）。
  - 群体共享：基于 GEP 协议（Genome Evolution Protocol）发布/订阅经验，自然选择优胜策略。
  - 群体进化（GEA 范式）：以任务为单元组织 Agent 群体，群体内显式共享经验，推动整体能力跃迁。
  - 置信度评分：经验胶囊附带置信度，使用次数越多、成功率越高，传播越广。

### 4. 模型聚合与路由
- **问题**：大规模 Agent 需要调用不同能力/成本的模型，如何统一管理？
- **方案**：
  - 统一接口：通过 OpenAI 兼容 API 网关屏蔽底层差异。
  - 智能路由：根据任务类型（代码生成、长文档、高频简单等）动态选择最优模型。
  - 自动降级：主模型失败/超时时切换到备用。
  - 成本优化：跟踪各模型价格/性能，动态调整路由策略。
  - 多供应商支持：集成 OpenRouter、n1n.ai、阿里云等聚合服务或直连。

## 四、当前技术挑战与待解决问题

1. **记忆折叠算法**：如何设计折叠触发条件？折叠后的胶囊如何保证信息完整性？长期检索的准确性与延迟？
2. **技能沙盒**：WASM vs 容器？如何平衡安全与性能？
3. **GEP 协议实现**：分布式环境下经验胶囊的发布/订阅机制？如何防止低质量经验污染群体？
4. **模型路由策略**：任务分类的规则引擎设计？成本优化算法如何动态调整？
5. **总体架构协同**：各模块数据流向？性能瓶颈预测？如何保证百万级 Agent 下的稳定？
